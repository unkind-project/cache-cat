# 删除策略

> 这里均为基于moka进行二次开发

## LRU和TinyLfu删除

Raft算法本身的限制：为了性能，读取操作不经过Raft决议，并且和Raft决议可以同时运行，可能导致删除的数据并不准确。

当你访问一个已经在删除过程中的数据，你可能仍然能读到这条数据，但是数据最终还是会被删除，虽然发生的概率很小。

无论是LRU还是TinyLfu的删除都会由主节点发起决议进行删除，来保证数据迭代的确定性。

这里fork了moka的源码实现获取LRU和LFU中队列最后，也就是最不常被访问的数据。然后发起决议对这些数据进行删除。这里有可能发生，在删除决议进行之前，这个数据又被进行了读取，这种情况下数据仍然会被进行删除。
